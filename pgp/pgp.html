Introdução

Este documento visa prestar informações e definir orientações sobre o desenvolvimento de sistema que virá a suportar o processo de controle de ordens de serviço utilizado por fábricas de software, possivelmente contratadas a partir de registros de preço celebrados, que prestem os serviços de medição e desenvolvimento e manutenção de software utilizando tanto o Processo de Gerenciamento Ágil quanto o processo de Desenvolvimento de Software por Aquisição ou o Processo de Manutenção de Software por Aquisição.

 

Inicialmente, a estratégia será priorizar as funcionalidades que mais retornem valor para a organização nos processos de gestão de aquisição, fazendo com que o produto seja desenvolvido parcialmente (3 releases) e a demonstração dos ganhos sejam incontestáveis. Posteriormente, as demais funcionalidades poderão ser desenvolvidas se for do entendimento da alta administração que a continuidade do projeto trará ainda mais valor.

Linhas de Base

O Plano de Gerenciamento de Projeto presente no Processo Gerenciamento Ágil (PGA) possui uma estrutura de três documentos. São eles:

1.    Estrutura Analítica do Projeto (EAP);

2.    Cronograma;

3.    Plano de Gerenciamento das Comunicações;

Após a aprovação deste Plano de Gerenciamento do Projeto os documentos listados abaixo servirão de base para o projeto, não devem ser alterados sem o conhecimento e aprovação das partes interessadas.
